PriorityClass — Scheduling Priority & Preemption

How Kubernetes decides which pending pods matter most, and when a high-priority pod can evict lower-priority pods to make room.

One sentence definition

A PriorityClass gives pods an integer priority; higher-priority pods are scheduled first and may preempt lower-priority pods if the cluster has no room.

The Mental Model

PriorityClass is not a resource request, not a node selector, and not a guarantee. It is a ranking. When multiple pods are waiting, the scheduler looks at higher-priority pods first. If a high-priority pod cannot fit anywhere, Kubernetes may remove lower-priority pods from a node so the important pod can land.

analogy · emergency lane

Normal pods wait in traffic. Critical pods have sirens. The siren does not create a bigger road, but it can make less important traffic move out of the way.

PriorityClass YAML

apiVersion: scheduling.k8s.io/v1
kind: PriorityClass
metadata:
  name: high-priority
value: 100000
globalDefault: false
preemptionPolicy: PreemptLowerPriority
description: "Important application pods"
FieldMeaning
valueThe integer priority. Higher number means higher priority.
globalDefaultIf true, pods without priorityClassName get this class by default.
preemptionPolicyPreemptLowerPriority by default, or Never to avoid preemption.
descriptionHuman-readable purpose.

Using It in a Pod

apiVersion: v1
kind: Pod
metadata:
  name: important-nginx
spec:
  priorityClassName: high-priority
  containers:
    - name: nginx
      image: nginx

Once admitted, the pod gets a numeric priority copied from the PriorityClass.

kubectl get pod important-nginx -o yaml | grep -n "priority"

What Preemption Means

If a high-priority pod is pending because no node has enough free resources, the scheduler checks whether removing lower-priority pods would make room. If yes, those lower-priority pods are marked for deletion, and the high-priority pod schedules after the space clears.

important

Preemption is not instant magic. Victim pods get graceful termination. The high-priority pod may remain Pending briefly while those pods exit.

Preemption Policy

PolicyBehavior
PreemptLowerPriority Default. Pod can evict lower-priority pods if needed.
Never Pod gets scheduling priority but will not preempt other pods. 
apiVersion: scheduling.k8s.io/v1
kind: PriorityClass
metadata:
  name: queue-jumper-no-evict
value: 50000
preemptionPolicy: Never
globalDefault: false
description: "Schedules before normal pods, but does not evict them"

Built-in Priority Classes

Kubernetes has special built-in classes for system-critical pods.

PriorityClassUse
system-cluster-critical Cluster-level critical components.
system-node-critical Node-critical components. Usually even higher priority.
✓ Exam shortcut — For custom apps, create your own PriorityClass. Do not casually assign system-critical classes unless the question explicitly says so.

Useful Commands

# List priority classes
kubectl get priorityclass
kubectl get pc

# Describe one
kubectl describe priorityclass high-priority

# See pod priority
kubectl get pod important-nginx -o yaml | grep -n "priority"

# See pending/preemption events
kubectl describe pod important-nginx

# Create from file
kubectl apply -f priorityclass.yaml

Imperative Quick Commands for CKA Exam

There is no direct kubectl run --priority-class flag. Use --overrides or generate YAML and edit it.

# Create a PriorityClass imperatively from inline YAML
kubectl create priorityclass high-priority \
  --value=100000 \
  --description="Important application pods"

# Create a Pod with priorityClassName using overrides
kubectl run important-nginx \
  --image=nginx \
  --overrides='{"spec":{"priorityClassName":"high-priority"}}'

# Dry-run version: create YAML first, then apply
kubectl run important-nginx \
  --image=nginx \
  --overrides='{"spec":{"priorityClassName":"high-priority"}}' \
  --dry-run=client -o yaml > pod-priority.yaml
kubectl apply -f pod-priority.yaml

# Verify the pod got numeric priority
kubectl get pod important-nginx -o jsonpath='{.spec.priorityClassName}{" "}{.spec.priority}{"\n"}'

Common Mistakes

✗ Mistake 1 — Putting priorityClassName under the container.
It belongs in the pod spec: spec.priorityClassName.
✗ Mistake 2 — Thinking priority ignores resource requests.
The pod still needs CPU, memory, node selectors, taints/tolerations, affinity, and other scheduling rules to work.
✗ Mistake 3 — Expecting preemption to evict equal-priority pods.
Preemption targets lower-priority pods, not pods with the same or higher priority.
✗ Mistake 4 — Forgetting apiVersion: scheduling.k8s.io/v1.
PriorityClass is not v1 and not apps/v1.
