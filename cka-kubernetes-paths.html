Kubernetes Paths — Files You Must Know

A fast map of where kubeconfigs, static pod manifests, certs, kubelet config, etcd data, logs, and systemd units usually live.

One sentence definition

In CKA troubleshooting, half the job is knowing which file to inspect: kubeadm clusters keep most important control-plane files under /etc/kubernetes and kubelet state under /var/lib/kubelet.

The Big Map

PathWhat lives thereWhen you use it
/etc/kubernetes/manifests Static pod manifests: kube-apiserver.yaml, kube-scheduler.yaml, kube-controller-manager.yaml, etcd.yaml Fix control-plane pods, edit API server flags, stop/restart static pods.
/etc/kubernetes/admin.conf Admin kubeconfig Use kubectl as cluster admin.
/etc/kubernetes/kubelet.conf Kubelet kubeconfig Kubelet's client credentials to talk to the API server.
/etc/kubernetes/controller-manager.conf Controller manager kubeconfig Control-plane component authentication.
/etc/kubernetes/scheduler.conf Scheduler kubeconfig Scheduler authentication to API server.
/etc/kubernetes/pki Kubernetes certificates and keys Check certs, CA files, API server cert paths.
/var/lib/kubelet/config.yaml Kubelet config file Find staticPodPath, cluster DNS, cgroup driver, auth settings.
/var/lib/etcd Default etcd data directory Backups, restores, data-dir checks.
/var/lib/kubelet/pods Kubelet's local pod state Deep node troubleshooting; mounted volumes and pod directories.
~/.kube/config User kubeconfig Your normal shell's kubectl config.

Control Plane Manifests

In kubeadm clusters, the control plane is usually Static Pods. The kubelet watches this directory:

/etc/kubernetes/manifests
FileComponentCommon reason to open it
kube-apiserver.yamlAPI serverAdd/edit admission plugins, cert paths, authorization flags, etcd connection.
kube-controller-manager.yamlController managerCluster CIDR, controllers, service account signing flags.
kube-scheduler.yamlSchedulerScheduler config, profiles, policy-related troubleshooting.
etcd.yamletcdData directory, certs, listen/client URLs.
careful

Saving a file in /etc/kubernetes/manifests makes kubelet restart that static pod. A bad edit to kube-apiserver.yaml can make kubectl stop working until you fix the local file.

Kubeconfig Locations

A kubeconfig is the file that tells a client how to reach the API server and which credentials to use.

PathWho uses itNotes
~/.kube/configHuman userDefault path for kubectl.
/etc/kubernetes/admin.confCluster adminOften copied to ~/.kube/config.
/etc/kubernetes/kubelet.confKubeletLets kubelet authenticate to the API server.
/etc/kubernetes/controller-manager.confController managerUsed by the static pod command.
/etc/kubernetes/scheduler.confSchedulerUsed by the static pod command.
# Use admin kubeconfig directly
kubectl --kubeconfig=/etc/kubernetes/admin.conf get nodes

# Make it your default user kubeconfig
mkdir -p ~/.kube
sudo cp /etc/kubernetes/admin.conf ~/.kube/config
sudo chown $(id -u):$(id -g) ~/.kube/config

Certificates

Most kubeadm certs live under /etc/kubernetes/pki.

PathMeaning
/etc/kubernetes/pki/ca.crtCluster CA certificate.
/etc/kubernetes/pki/ca.keyCluster CA private key. Very sensitive.
/etc/kubernetes/pki/apiserver.crtAPI server serving certificate.
/etc/kubernetes/pki/apiserver.keyAPI server private key.
/etc/kubernetes/pki/front-proxy-ca.crtFront proxy CA.
/etc/kubernetes/pki/etcdetcd CA, server, peer, and client certs.
# Check certificate expiry
kubeadm certs check-expiration

# Inspect one cert
openssl x509 -in /etc/kubernetes/pki/apiserver.crt -text -noout

Kubelet Paths

PathWhat it is
/var/lib/kubelet/config.yamlMain kubelet config. Look for staticPodPath.
/var/lib/kubelet/kubeadm-flags.envKubeadm-managed kubelet flags.
/var/lib/kubelet/podsLocal pod data directories.
/var/lib/kubelet/pluginsCSI and kubelet plugin data.
# Find static pod path
sudo grep -n "staticPodPath" /var/lib/kubelet/config.yaml

# Read kubelet service config
systemctl cat kubelet

# Follow kubelet logs
journalctl -u kubelet -f

etcd Paths

For kubeadm, etcd usually runs as a Static Pod and stores data here:

/var/lib/etcd

The exact value is visible in the etcd manifest:

sudo grep -n "data-dir" /etc/kubernetes/manifests/etcd.yaml
✓ Exam shortcut — For backup/restore questions, inspect /etc/kubernetes/manifests/etcd.yaml for endpoints, cert paths, and --data-dir. Do not rely only on memory.

Common Mistakes

✗ Mistake 1 — Editing the mirror pod with kubectl edit.
For static pods, edit the file in /etc/kubernetes/manifests.
✗ Mistake 2 — Forgetting that admin.conf is not the kubelet's kubeconfig.
Humans use admin.conf; kubelet uses /etc/kubernetes/kubelet.conf.
✗ Mistake 3 — Searching random places for static pods.
Read staticPodPath from /var/lib/kubelet/config.yaml. In kubeadm it is usually /etc/kubernetes/manifests.

Imperative Quick Commands for CKA Exam

# Use admin kubeconfig for one command
kubectl --kubeconfig=/etc/kubernetes/admin.conf get nodes

# Set default kubeconfig for your user
mkdir -p ~/.kube
sudo cp /etc/kubernetes/admin.conf ~/.kube/config
sudo chown $(id -u):$(id -g) ~/.kube/config

# Find static pod path and inspect manifests
sudo grep -n "staticPodPath" /var/lib/kubelet/config.yaml
sudo ls -l /etc/kubernetes/manifests

# Check kubelet and certs
systemctl cat kubelet
journalctl -u kubelet -f
kubeadm certs check-expiration
