Imperative Commands — One-Glance Cheat Sheet

Fast commands, dry-run patterns, and small output examples for the CKA. Use imperative commands to generate YAML, then edit only what the command cannot express.

The pattern

Most CKA speed comes from --dry-run=client -o yaml > file.yaml, then editing the few fields that kubectl cannot set with flags.

Imperative Quick Commands for CKA Exam

This whole page is the quick-command view. Start with the direct command; if a flag does not exist, use --overrides for small pod-spec changes or generate YAML with --dry-run=client -o yaml.

Core Creation Commands

Pod

kubectl run nginx \
  --image=nginx
pod/nginx created
kubectl run nginx \
  --image=nginx \
  --dry-run=client -o yaml > pod.yaml
# no terminal output
# pod.yaml file is created

Export Existing Pod

Use this when the pod already exists. No --dry-run needed.

kubectl get pod critical-app \
  -o yaml > crtapp.yaml
# no terminal output
# crtapp.yaml file is created

Deployment

kubectl create deployment nginx \
  --image=nginx \
  --replicas=3
deployment.apps/nginx created
kubectl create deployment nginx \
  --image=nginx \
  --dry-run=client -o yaml > deploy.yaml
# no terminal output
# deploy.yaml file is created

Service

kubectl expose pod nginx \
  --port=80 \
  --target-port=80 \
  --name=nginx-svc
service/nginx-svc exposed
kubectl create service clusterip nginx \
  --tcp=80:80
service/nginx created

Namespace

kubectl create ns dev
kubectl config set-context --current --namespace=dev
namespace/dev created
Context "..." modified.

Resources

Requests / Limits

kubectl run nginx --image=nginx \
  --requests='cpu=100m,memory=128Mi' \
  --limits='cpu=500m,memory=256Mi'
pod/nginx created

Edit Generated YAML

kubectl run nginx --image=nginx --dry-run=client -o yaml > pod.yaml
vi pod.yaml
kubectl apply -f pod.yaml
# file generated, edited, then:
pod/nginx created

Jobs & CronJobs

Job

kubectl create job pi --image=perl -- perl -Mbignum=bpi -wle 'print bpi(2000)'
kubectl get jobs,pods
job.batch/pi created
NAME   COMPLETIONS   DURATION   AGE

CronJob

kubectl create cronjob hello \
  --image=busybox \
  --schedule="*/1 * * * *" \
  -- date
cronjob.batch/hello created

Troubleshooting Output Patterns

# Wide placement view
kubectl get pods -o wide

# Events and scheduling errors
kubectl describe pod nginx

# Logs
kubectl logs nginx
kubectl logs deploy/nginx

# Rollout
kubectl rollout status deploy/nginx
kubectl rollout history deploy/nginx
kubectl rollout undo deploy/nginx

# Explain fields
kubectl explain pod.spec
kubectl explain deployment.spec.template.spec.affinity
kubectl explain priorityclass
